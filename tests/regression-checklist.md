# Regression Checklist

## Tone

- [ ] Không còn xã giao dư kiểu `chắc chắn rồi`, `để mình giúp bạn nhé`.
- [ ] Không còn hedge thừa kiểu `mình nghĩ`, `có vẻ`, `có lẽ` khi không cần.
- [ ] Câu vẫn tự nhiên với dev Việt.
- [ ] `ultra` không thành tối nghĩa.

## Technical Fidelity

- [ ] Code block giữ nguyên 100%.
- [ ] Inline code giữ nguyên 100%.
- [ ] URL, path, env var, version giữ nguyên.
- [ ] Error message và stack trace không bị rewrite.
- [ ] Không dịch sai proper noun hoặc tên library/API.

## Skill Behavior

- [ ] `lite` vẫn là câu đầy đủ.
- [ ] `full` ngắn và trực diện hơn `lite`.
- [ ] `ultra` ngắn nhất nhưng vẫn đủ actor/action/reason khi cần.
- [ ] `gon-commit` không lan man.
- [ ] `gon-review` có problem + fix cụ thể.
- [ ] `gon-compress` không đụng code/config/data.

## Safety / Fallback

- [ ] destructive op chuyển sang rõ hơn
- [ ] security issue có context đủ
- [ ] migration nhiều bước có thứ tự rõ
- [ ] production/database/infra không bị trả lời kiểu shorthand nguy hiểm
- [ ] user hỏi lại nhiều lần thì output rõ hơn, không ngắn hơn

## Localization Quality

- [ ] Không có câu dịch literal từ English prompt style.
- [ ] Không có persona "hang động" giả trân.
- [ ] Có dùng đúng lexicon rút gọn tiếng Việt đã định nghĩa.
- [ ] Nếu dùng thuật ngữ tiếng Anh, đó là form dev Việt thực tế hay dùng.
